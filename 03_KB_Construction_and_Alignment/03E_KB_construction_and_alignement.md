# 03E — KB Construction and Alignment

The input of this step is the CSV file produced by the Information Extraction module:

2. RDF Modeling Choices

The extracted information is represented using RDF triples.

Each extracted fact has the form:

subject → predicate → object

For example:

triceps → extend → elbow
bar → apply → downward force
grip width → determine → horizontal flexion demands

In RDF, these are represented as:

ex:triceps ex:extends ex:elbow .
ex:bar ex:appliesForce ex:downward_force .
ex:grip_width ex:influences ex:horizontal_flexion_demands .

The ontology contains several entity classes:

Class	Meaning
Exercise	Exercises or movement patterns
Muscle	Muscles involved in the bench press
Joint	Joints and body parts
Equipment	Training equipment
TechniqueCue	Technique-related concepts
BiomechanicalConcept	Biomechanical concepts such as force, torque, moment arm, or activation
DomainEntity	Generic fallback class

The ontology also defines controlled predicates such as:

targetsMuscle
influences
appliesForce
requires
involves
extends
flexes
reduces
relatedTo
3. Entity Linking with Confidence

Entity linking maps different surface forms to the same canonical entity.

For example:

pecs → pectoralis major
pectorals → pectoralis major
chest → pectoralis major
barbell → bar
scapulae → scapula

This prevents the knowledge base from creating duplicate nodes for the same concept.

For example, without entity linking, the graph could contain separate nodes:

pecs
chest
pectoralis major

After alignment, these are represented as:

pectoralis major

Each extracted triple also keeps a confidence score from the Information Extraction step. This confidence is stored in a statement node.

Example:

ex:statement_001
    ex:hasSubject ex:triceps ;
    ex:hasPredicate ex:extends ;
    ex:hasObject ex:elbow ;
    ex:confidence "0.82" ;
    ex:evidenceSentence "The triceps help extend the elbow." ;
    ex:sourceDocument "nasm_bench_press.html" .

This allows the graph to keep both the clean RDF triple and the evidence metadata.

4. Predicate Alignment

Predicate alignment maps noisy extracted relations to a controlled vocabulary.

For example:

Extracted relation	Aligned predicate
activate	targetsMuscle
activation_of	targetsMuscle
target	targetsMuscle
determine	influences
affect	influences
apply	appliesForce
extend	extends
flex	flexes
require	requires
involve	involves
reduce	reduces

This is important because extracted text contains many different verbs that can express the same semantic relation.

For example:

bench press works the chest
bench press activates the pectoralis major
bench press targets the pecs

These can all be aligned to:

bench press → targetsMuscle → pectoralis major
5. Expansion Strategy

After building the initial KB, the graph is expanded using simple SPARQL CONSTRUCT rules.

The goal is to add higher-level semantic relations from existing triples.

Rule 1 — Muscle acts on joint

If a muscle flexes, extends, abducts, or adducts a joint, add:

muscle → actsOnJoint → joint

Example:

triceps → extends → elbow

Expansion:

triceps → actsOnJoint → elbow
Rule 2 — Technique cue affects biomechanics

If a technique cue influences, increases, decreases, or reduces a biomechanical concept, add:

technique cue → affectsBiomechanics → biomechanical concept

Example:

grip width → influences → range of motion

Expansion:

grip width → affectsBiomechanics → range of motion
Rule 3 — Equipment involves biomechanical concept

If equipment applies force, add:

equipment → involvesBiomechanicalConcept → force

Example:

bar → appliesForce → downward force

Expansion:

bar → involvesBiomechanicalConcept → downward force
Rule 4 — Generic related concept

If an entity requires or involves another entity, add:

entity → relatedTo → entity

This helps connect concepts that are related but may not fit a more specific predicate.

8. Reflection

This KB construction step improves the raw extraction output in several ways.

First, RDF modeling makes the extracted facts machine-readable and queryable.

Second, entity linking reduces duplicate entities by aligning different mentions such as pecs, chest, and pectoralis major.

Third, predicate alignment reduces noise by mapping many extracted verbs to a smaller controlled predicate vocabulary.

Fourth, confidence scores and evidence sentences are preserved so the KB does not lose the original extraction context.

Finally, expansion rules enrich the graph by adding useful higher-level relations such as actsOnJoint and affectsBiomechanics.

The main limitation is that the alignment is still rule-based. It works well for a small controlled domain, but a larger project would benefit from more advanced entity linking and ontology matching methods.
