3. NL→SPARQL Prompt Template

The system uses the following prompt template concept:

You are an RDF/SPARQL assistant for a bench press biomechanics knowledge graph.

Task:
Convert the user question into a SPARQL query.

Rules:
1. Use only the namespace ex: <http://example.org/benchpress-kg/>.
2. Use only known ontology classes and predicates.
3. Return only a valid SPARQL SELECT query.
4. Prefer specific predicates such as:
   - ex:targetsMuscle
   - ex:actsOnJoint
   - ex:affectsBiomechanics
   - ex:appliesForce
   - ex:extends
   - ex:flexes
   - ex:requires
   - ex:influences
   - ex:relatedTo
5. If the question is broad, return a query that searches connected triples.

In this implementation, the NL→SPARQL step is rule-based rather than dependent on an external LLM. This keeps the demo reproducible and avoids API dependencies.

4. Self-Repair Mechanism

The RAG system includes a simple self-repair mechanism.

If the generated SPARQL query fails, the system attempts to repair:

missing prefixes
predicate casing errors
common predicate name mismatches

Examples:

ex:appliesforce → ex:appliesForce
ex:targetsmuscle → ex:targetsMuscle
ex:affectsbiomechanics → ex:affectsBiomechanics
ex:actsonjoint → ex:actsOnJoint

If the repaired query still returns no answer, the system uses a broader fallback query.

The fallback query searches project KB triples using keywords detected in the original question.

This means the system can still provide a useful answer even when the first SPARQL query is too specific.

5. Demo

The interactive demo is run with:

python .\06_RAG_over_RDF_SPARQL\06B_rag_sparql_demo.py

Example question:

Which technique cues affect biomechanical concepts?

Generated SPARQL:

PREFIX ex: <http://example.org/benchpress-kg/>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?cueLabel ?conceptLabel WHERE {
    ?cue a ex:TechniqueCue .
    ?concept a ex:BiomechanicalConcept .
    ?cue ex:affectsBiomechanics ?concept .
    OPTIONAL { ?cue rdfs:label ?cueLabel . }
    OPTIONAL { ?concept rdfs:label ?conceptLabel . }
}

Expected answer:

grip width affects horizontal flexion demands

7. Reflection

The RAG/SPARQL system provides grounded answers from the RDF knowledge base.

Compared with a generic baseline, the RAG system can return exact structured facts from the graph.

For example:

grip width affectsBiomechanics horizontal flexion demands
triceps actsOnJoint elbow
bar appliesForce downward force

The main advantage is that the answers are traceable to the structured KB.

The main limitation is that the NL→SPARQL translation is currently rule-based. A future version could use an LLM to generate SPARQL dynamically from the schema summary, then apply the same self-repair mechanism.
